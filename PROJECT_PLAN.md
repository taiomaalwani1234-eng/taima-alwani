## المرحلة الثالثة: ترقية محاكاة "مدينة آمنة" (اللعب الفردي التكتيكي) 🚀 (الهدف الحالي)
سيتم تحويل اللعبة لتصبح لعبة "إدارة موارد تكتيكية" تواجه فيها المدينة هجمات متصاعدة:
1. **الميكانيكا الأساسية**: 
   - اللاعب يبدأ برصيد مالي (مثلاً: 50,000$).
   - تتعرض المدينة لهجوم على عقدة محددة.
   - يتطلب اختيار قرار دفاعي استنزاف الرصيد (الإجابة الصحيحة تكلف مبلغاً معقولاً، الإجابة الخاطئة تستنزف الرصيد بشدة وتزيد الضرر).
2. **المستوى الأول (أساسي)**: خريطة بسيطة (6 عقد). كل هجوم يعرض **5 خيارات دفاعية**. تجاوز المستوى يتطلب صد 5 هجمات بنجاح.
3. **المستوى الثاني (متقدم)**: تنتقل المدينة لشبكة معقدة. كل هجوم يوفر **10 خيارات دفاعية** متقاربة جداً مما يرفع نسبة التحدي.
4. **المستوى الثالث (كابوس سيبراني)**: تتوسع خريطة المدينة لتشمل **25 منصة وعقدة**. يتعرض اللاعب لهجمات متزامنة مع وجود **12 خياراً دفاعياً** لكل موقف، محاكيةً أقصى درجات الضغط والتعقيد الهندسي.

## المرحلة الرابعة: نظام اللعب الجماعي (Multiplayer) المتقدم (مخطط لاحقاً)
لعبة محاكاة سيبرانية تنافسية (فريق ضد فريق) لتعزيز العمل الجماعي:
1. **الخوادم والبنية (Firebase)**: إعداد قواعد البيانات السحابية (Firestore) وخدمات المصادقة (Auth) للحفظ والتحديث المتزامن لحظياً (Real-time).
2. **الغرف والتوفيق (Lobbies)**: واجهة يمكن للطلاب من خلالها إنشاء غرفة (Room) أو الانضمام إليها عبر رمز سري.
3. **أدوار الفرق**: 
   - **الفريق الأحمر (Red Team)**: ميزانية لاختراق البنية التحتية، شراء فيروسات، DDoSing، أو هندسة اجتماعية.
   - **الفريق الأزرق (Blue Team)**: ميزانية لصد الهجمات، ترقية الجدران النارية، واكتشاف الثغرات (Patching).
4. **نظام النقاط والفوز**: واجهة تعرض سيطرة الفريق الأحمر مقابل سرعة استجابة الفريق الأزرق ضمن مدة زمنية محددة.

## المرحلة الخامسة: لعبة "اختراق الخادم" — محاكاة طرفية SSH 🖥️ (جديدة)

لعبة تعليمية تفاعلية تحاكي بيئة Terminal حقيقية، حيث يُطلب من المتدرب اختراق خادم عبر SSH باستخدام الأوامر الصحيحة، مع نظام حماية fail2ban يحظر المحاولات الفاشلة.

### 1. المكوّن الجديد: `SSHGameView.tsx`

يُضاف كـ view جديد في `App.tsx` مع ViewState = `'ssh'` ويظهر كبطاقة في لوحة التحكم (Dashboard).

### 2. واجهة اللعبة — شاشة البداية (Mission Briefing)

- تُعرض شاشة إحاطة عسكرية بتصميم أخضر على أسود (نمط Terminal كلاسيكي).
- تتضمن:
  - **عنوان IP الهدف**: يتم توليده عشوائياً لكل جلسة لعب (مثل `192.168.45.12`).
  - **المستخدم المطلوب**: `root`.
  - **المهمة**: "اخترق الخادم وتفحّص ملفاته السرية".
  - زر **"ابدأ المهمة"** ينتقل إلى واجهة Terminal التفاعلية.

### 3. واجهة Terminal التفاعلية

- **تصميم**: خلفية سوداء `#0a0a0a`، خط `monospace` أخضر (`#00ff41`)، مؤشر وميض (blinking cursor).
- **الـ Prompt**: `hacker@kali:~$` (قبل تسجيل الدخول) → `root@target:~#` (بعد تسجيل الدخول بنجاح).
- **حقل إدخال**: input يُحاكي سطر أوامر حقيقي (الضغط على Enter لتنفيذ الأمر).
- **سجل الأوامر (history)**: يُعرض في div قابل للتمرير كل أمر ومخرجاته.

### 4. مراحل اللعبة

#### المرحلة A — الاتصال بالخادم
| إدخال المستخدم | الاستجابة |
|---|---|
| `ssh root@<IP_الصحيح>` | `root@<IP>'s password:` — ينتقل لوضع إدخال كلمة السر (يُخفى النص كـ `***`) |
| أي أمر آخر | رسالة خطأ واقعية: `bash: <command>: command not found` أو `ssh: connect to host <x> port 22: Connection refused` حسب السياق |
| `ssh` بدون معاملات | `usage: ssh [-46AaCfGgKkMNnqsTtVvXxYy] [-B bind_interface] ...` |
| `ssh root@<IP_خاطئ>` | `ssh: connect to host <IP> port 22: Connection timed out` |

#### المرحلة B — إدخال كلمة السر
| إدخال المستخدم | الاستجابة |
|---|---|
| `root` (الكلمة الصحيحة) | رسالة ترحيب: `Welcome to Ubuntu 22.04.3 LTS ...` + `Last login: ...` ← ينتقل لمرحلة ما بعد الدخول |
| أي كلمة أخرى | `Permission denied, please try again.` — عداد المحاولات +1 |
| فشل 3 مرات | `Permission denied (publickey,password).` ← ينقله لشاشة الحظر (fail2ban) |

#### المرحلة C — بيئة ما بعد الدخول (Post-Login Shell)
بعد نجاح تسجيل الدخول، يتغير الـ prompt إلى `root@target:~#` ويمكن للمتدرب تنفيذ أوامر محاكاة:

| الأمر | المخرجات |
|---|---|
| `ls` | `secret.txt  passwords.db  .bash_history  network_config  logs/` |
| `cat secret.txt` | `FLAG{congratulations_you_hacked_the_server}` + رسالة نجاح |
| `cat passwords.db` | جدول مُنسق بأسماء مستخدمين وهاشات كلمات سر وهمية |
| `cat .bash_history` | سجل أوامر وهمي يتضمن `wget`, `chmod`, `nmap` |
| `ip a` أو `ifconfig` | معلومات شبكة وهمية (IP الخادم، subnet، MAC address) |
| `whoami` | `root` |
| `uname -a` | `Linux target 5.15.0-91-generic #101-Ubuntu SMP ...` |
| `pwd` | `/root` |
| `cd logs && ls` | `auth.log  syslog  access.log` |
| `cat logs/auth.log` | محاولات تسجيل الدخول الفاشلة (تتضمن IP اللاعب الحقيقي!) |
| `hostname` | `target-server` |
| `id` | `uid=0(root) gid=0(root) groups=0(root)` |
| `exit` | يخرج من الجلسة ويعود للـ prompt الأصلي أو ينهي اللعبة |
| أي أمر غير معروف | `bash: <command>: command not found` |

### 5. نظام الحظر — محاكاة fail2ban

#### آلية العمل:
- **تسجيل IP اللاعب الحقيقي**: عند بدء اللعبة يُحفظ IP اللاعب في `localStorage` (أو يُستجلب من API خارجي بسيط).
- **عدّاد المحاولات**: يُخزَّن في `localStorage` مرتبطاً بالـ IP:
  ```
  ssh_attempts_{IP} = { count: number, bannedAt: timestamp | null }
  ```
- **الحظر**: عند الوصول لـ 3 محاولات فاشلة:
  - يتم حظر الـ IP ومنعه من إعادة المحاولة.
  - تظهر شاشة حظر بتصميم أحمر تحذيري.
  
#### شاشة الحظر (Ban Screen):
- رسالة: `⚠️ تم حظر عنوانك [IP] بواسطة fail2ban بسبب 3 محاولات فاشلة.`
- تظهر **صفحة فك الحظر** داخل اللعبة نفسها:
  - يتضمن **لوحة إدارة fail2ban** مُحاكاة.
  - يعرض جدولاً بالعناوين المحظورة (يتضمن IP اللاعب الحقيقي).
  - زر **"فك الحظر"** بجانب كل IP.
  - عند الضغط عليه: يُطلب من اللاعب كتابة الأمر الصحيح:
    ```
    fail2ban-client set sshd unbanip <IP>
    ```
  - إذا أدخل الأمر الصحيح → يُزال الحظر ويعود لمحاولة الاتصال من جديد.
  - إذا أدخل أمراً خاطئاً → رسالة: `Error: Invalid command syntax. Try: fail2ban-client set <jail> unbanip <IP>`

### 6. نظام النقاط والتقييم

| الإنجاز | النقاط |
|---|---|
| الاتصال بنجاح عبر SSH من أول محاولة | +300 |
| إدخال كلمة السر الصحيحة من أول مرة | +200 |
| قراءة ملف `secret.txt` (العلم/FLAG) | +500 |
| استكشاف 3 أوامر أو أكثر بعد الدخول | +100 |
| فك حظر fail2ban بنجاح | +150 |
| **خصم**: كل محاولة كلمة سر فاشلة | -50 |

### 8. ملاحظات تقنية

- **لا حاجة لـ backend إضافي**: كل المنطق يعمل في الـ frontend (React state + localStorage).
- **الـ IP الحقيقي**: يمكن جلبه من `https://api.ipify.org?format=json` أو استخدام IP وهمي ثابت كـ fallback.
- **الأمان**: اللعبة محاكاة تعليمية بالكامل — لا اتصال حقيقي بأي خادم.
- **Responsive**: الواجهة تعمل على الموبايل (Terminal يتكيف مع حجم الشاشة).
- **إعادة اللعب**: زر "إعادة المهمة" يُنشئ IP عشوائي جديد ويُصفّر المحاولات.
